GraphQL サポート

このアドオンを使用すると、GraphQL定義をインポートし、その定義から生成されたクエリを送信できます。

アドオンは、スコープ内にある限り、GraphQL定義を自動的に検出してスパイダー処理を行います。
このスパイダー機能はZAP 2.10.0以降でサポートされています。

UI

「インポート」メニューに以下の項目が追加されます：
  • GraphQLスキーマのインポート

エンドポイントURLの形式

エンドポイントURLは次の形式を取ります：
scheme://authority/path
ファイルからインポートする場合、URIの全コンポーネントが必須となります。

オプションパネル

ツール -> オプション -> GraphQLの下にGraphQLオプションパネルが追加されます。 これらのオプションを使用して、クエリジェネレーターの出力を制御できます。

API

APIに以下の操作が追加されます：

ビュー

  • optionArgsType
  • optionMaxArgsDepth
  • optionMaxQueryDepth
  • optionOptionalArgsEnabled
  • optionQuerySplitType
  • optionQueryGenEnabled
  • optionRequestMethod

アクション

  • importFile (endurl*, file*)
  • importUrl (endurl*, url)
  • setOptionArgsType (String*)
  • setOptionMaxArgsDepth (Integer*)
  • setOptionMaxQueryDepth (Integer*)
  • setOptionOptionalArgsEnabled (Boolean*)
  • setOptionQueryGenEnabled (Boolean*)
  • setOptionQuerySplitType (String*)
  • setOptionRequestMethod (String*)
endurl は、前述のエンドポイントURL形式をサポートします。 定義は同期的にインポートされ、警告は返されます。

コマンドライン

以下のコマンドラインオプションが追加されます：
  • -graphqlfile <filename> : ファイルからGraphQL定義をインポートします
  • -graphqlurl <url> : URLからGraphQL定義をインポートします
  • -graphqlendurl <url> : エンドポイントURLを設定します
定義は同期的にインポートされ、警告があればコマンドラインに表示されます。
