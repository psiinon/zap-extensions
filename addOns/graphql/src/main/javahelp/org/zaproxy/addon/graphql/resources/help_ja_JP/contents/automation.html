GraphQL – 自動化フレームワーク対応

このアドオンは自動化フレームワークに対応しています。

スパイダー実行時にエンドポイントが発見された場合、このアドオンはイントロスペクションを利用してGraphQLスキーマをインポートします。 ただし、エンドポイントと (オプションで) スキーマファイルまたはURLが利用可能な場合は、それらを明示的に指定することを推奨します。

ジョブ: graphql

graphql ジョブを使用すると、ローカルファイルまたはURLからGraphQLスキーマをインポートできます。 以下のパラメーターをサポートしています：

この内容は、動画「ZAP Chat 11 Automation Framework Part 5 - APIs」で解説されています。 

  - type: graphql                      # GraphQL定義のインポート
    parameters:
      endpoint:                        # 文字列: エンドポイントのURL。デフォルト: null。この場合、スキーマはインポートされません
      schemaUrl:                       # 文字列: GraphQLスキーマを指すURL。デフォルト: null。エンドポイントへのイントロスペクションを通じてインポートされます
      schemaFile:                      # 文字列: GraphQLスキーマのローカルファイルパス。デフォルト: null。schemaUrlが指定されていない場合、ここからインポートされます
      queryGenEnabled:                 # Bool: クエリジェネレーターが有効かどうか。デフォルト: true
      maxQueryDepth:                   # Int: クエリ生成の最大深度。デフォルト: 5
      lenientMaxQueryDepthEnabled:     # Bool: 最大クエリ深度の緩和が有効かどうか。デフォルト: true
      maxAdditionalQueryDepth:         # Int: 最大追加クエリ深度 (緩和が有効な場合に使用)。デフォルト: 5
      maxArgsDepth:                    # Int: 最大引数生成深度。デフォルト: 5
      optionalArgsEnabled:             # Bool: オプション引数を指定するかどうか。デフォルト: true
      argsType:                        # Enum [inline, variables, both]: 引数の指定方法。デフォルト: both
      querySplitType:                  # Enum [leaf, root_field, operation]: 単一クエリが生成されるレベル。デフォルト: leaf
      requestMethod:                   # Enum [post_json, post_graphql, get]: リクエストメソッド。デフォルト: post_json
