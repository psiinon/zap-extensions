GraphQL オプション

本ドキュメントにおける「クエリ」は、GraphQLのクエリ、サブスクリプション、またはミューテーションを指すことがあります。

クエリジェネレーター設定

クエリジェネレーターは、インポートされたスキーマを使用して、ターゲットエンドポイント用のクエリを生成します。 有効になっている場合、以下のオプションで設定できます。

最大クエリ深度

生成されるクエリの最大深度。

寛容な最大クエリ深度

有効にすると、このオプションは、リーフタイプに到達していないフィールドに対して追加の深度を許可することで、無効なクエリを防止します。 無効にすると、最大クエリ深度は厳密に適用され、その結果、無効なクエリが生成される可能性があります。

追加クエリ深度

このオプションの値は、「寛容な最大クエリ深度」オプションが有効な場合にのみ使用されます。 リーフタイプフィールドを見つけるために使用される追加の最大深度。 リーフタイプが見つからない場合、生成されたクエリは無効になる可能性があり、メッセージがログに記録されます。

最大引数深度

指定される引数の最大深度。 これは、フィールドが引数として入力オブジェクトを持つ場合に役立ちます。

オプション引数を指定

このオプションが選択されている場合、オプション (null 許容) の引数も指定されます。

引数の指定方法

生成されたクエリにフィールド引数を追加する方法を選択します。
  • インライン
  • 変数を使用
  • 両方 - 各クエリは、インライン引数を使用する場合と変数を使用する場合の2回送信されます。

クエリの生成対象

提供されたスキーマ内で、以下のいずれかに対して個別のクエリが生成されます。
  • 各リーフ、すなわち、スカラーまたはEnum (列挙型)
  • ルート操作タイプ (Query、Mutation、または Subscription) の下にある各フィールド
  • 各ルート操作タイプ

リクエストメソッド

エンドポイントに対して行われるリクエストは、以下のタイプです。
  • JSONボディを持つPOSTリクエスト (Content-Type: application/json)
  • ボディにGraphQLクエリを含むPOSTリクエスト (Content-Type: application/graphql)
  • クエリがクエリ文字列として付加されたGETリクエスト
