自動化フレームワーク対応

このアドオンは自動化フレームワークに対応しています。

ジョブ: import

インポートジョブを使用すると、HAR (HTTPアーカイブファイル)、ModSecurity2ログ、ZAPメッセージ、またはURLを含むローカルファイルをインポートできます。 
  - type: import                       # データを含むファイルをインポートします
    parameters:
      type:                            # 文字列: ['har', 'modsec2', 'url', 'zap_messages'] のいずれか
      fileName:                        # 文字列: データを含むファイルの名前

ジョブ: export

エクスポートジョブを使用すると、メッセージをHAR形式またはURLとしてもエクスポートできるほか、サイトツリー形式でサイトツリーをエクスポートできます。 サポートされているソースは次のとおりです：
  • all: ZAPによって生成されたものを含むすべてのメッセージ。'har' および 'url' をサポート
  • history: 手動でまたはプロキシ経由で取得されたメッセージ。'har' および 'url' をサポート
  • sitestree: ZAPサイトツリー。'yaml' をサポート 
  - type: export            # データをファイルにエクスポートします
      parameters:
        context:            # 文字列: エクスポート元のコンテキストの名前。 デフォルト: 最初のコンテキスト
        type:               # 文字列: 'har'、'url'、'yaml'のいずれか。 デフォルト: 'har'
        source:             # 文字列: 'history'、'sitestree'、'all' のいずれか。 デフォルト: 'history'
        fileName:           # 文字列: ファイルの名前/パス

ジョブ: prune

prune ジョブを使用すると、ファイルからのデータを使用してサイトツリーからノードを剪定 (削除) できます。 ファイルはサイトツリー形式を使用する必要があります。 
  - type: prune           # ファイルからのサイトツリーデータ (YAML) を使用して、サイトツリーからノードを剪定します
    parameters:
      fileName:           # 文字列: ファイルの名前/パス
