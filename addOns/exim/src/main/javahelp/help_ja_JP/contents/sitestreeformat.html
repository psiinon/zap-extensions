サイトツリーファイル形式

サイトツリー形式は、ZAPサイトツリーのYAML表現です。
これはノードの階層であり、各ノードは、サイトツリー内の対応するノードを一意に識別するために必要なすべての本質的な情報を表します。

各ノードは次の形式を持っています： 

  - node:              # ZAPサイトツリーに表示されるノードの名前
    url:               # それが表すURL。トップノード以外のすべてに存在する
    method:            # HTTPメソッド。トップノード以外のすべてに存在する
    responseLength:    # 該当する場合、レスポンスの長さ
    statusCode:        # 該当する場合、HTTPステータスコード
    data:              # 存在する場合、データパラメータの名前。'=&' とアンパサンド (&) の組み合わせで区切られます
    children:          # 子ノードのリスト。リーフノードを除くすべてのノードに存在する

この形式は、自動化フレームワークの export および prune ジョブ、ならびに対応するデスクトップメニューで使用されます。

完全な単純な例： 

- node: Sites
  children:
  - node: https://www.example.com
    url: https://www.example.com
    method: GET
    children:
    - node: missing
      url: https://www.example.com/missing
      method: GET
      responseLength: 1221
      statusCode: 404
    - node: path
      url: https://www.example.com/path
      method: GET
      responseLength: 1234
      statusCode: 200
      children:
      - node: GET:query(q)
        url: https://www.example.com/seq/query?q=search
        method: GET
        responseLength: 2345
        statusCode: 200
      - node: submit
        url: https://www.example.com/seq/submit()(field1,field2,field3)
        method: POST
        data: field1=&field2=&field3=
        responseLength: 3456
        statusCode: 200
